# PR 리뷰 코멘트 확인 및 수정

현재 브랜치에 올라가 있는 PR의 리뷰 코멘트를 확인하고, 사용자가 선택한 항목을 수정합니다.

## 실행 흐름

### 1. PR 정보 확인

```bash
gh pr view --json number,title,url,state,reviewDecision
```

- PR이 없으면 중단: "현재 브랜치에 열린 PR이 없습니다."
- PR 번호, 제목, 상태, 리뷰 결과를 표시

### 2. 리뷰 코멘트 수집

```bash
# owner/repo 추출
gh repo view --json owner,name --jq '.owner.login + "/" + .name'

# 리뷰 코멘트 조회
gh api repos/{owner}/{repo}/pulls/{number}/comments
```

코멘트를 두 그룹으로 분리:
- **사람 리뷰어**: 봇이 아닌 코멘트 (github-actions[bot], coderabbitai[bot] 등 제외)
- **봇 코멘트**: 자동 생성된 코멘트 (요약만 표시)

### 3. 코멘트 정리 및 테이블 제시

사람 리뷰어 코멘트를 번호를 매겨 테이블로 정리:

```
## 사람 리뷰어 코멘트 - 총 N개

| # | 파일 | 라인 | 리뷰어 | 내용 |
|---|------|------|--------|------|
| 1 | Component.vue | 75 | reviewer1 | :deep() 문법 사용해주세요 |
| 2 | Component.vue | 58 | reviewer1 | === 로 변경해주세요 |

## 봇 코멘트 요약
- 요약1
- 요약2
```

### 4. 사용자에게 수정할 항목 선택 요청

사용자에게 어떤 번호를 수정할지 물어봅니다.
선택하지 않은 항목은 건드리지 않습니다.

### 5. 선택된 항목 수정

선택된 코멘트에 대해 하나씩 진행:

1. 해당 파일을 **반드시 먼저 Read**
2. 코멘트 내용에 맞게 수정
3. **변경 전/후 diff를 사용자에게 보여줌**
4. 사용자 컨펌 후 다음 항목으로 진행

코멘트 유형별 처리:
- **코드 수정 요청**: 직접 수정하고 diff 제시
- **질문/스펙 확인**: "이 항목은 PR 코멘트로 답변이 필요합니다"라고 안내
- **Nitpick/제안**: 수정하고 diff 제시

### 6. 마무리

모든 수정 완료 후:

1. 변경된 파일 목록 정리
2. 커밋 여부를 사용자에게 확인
3. 커밋 시 메시지 형식: `fix: 코드리뷰 반영`
4. push 여부도 확인

## 에러 처리

| 상황 | 대응 |
|------|------|
| PR이 없음 | "현재 브랜치에 열린 PR이 없습니다" |
| 코멘트가 없음 | "리뷰 코멘트가 없습니다" |
| gh CLI 미설치 | "gh CLI가 필요합니다: brew install gh" |
| 파일을 찾을 수 없음 | 사용자에게 경로 확인 요청 |

## 사용 예시

```
사용자: 리뷰 반영해줘
AI: PR #123 확인 - CHANGES_REQUESTED 상태

## 사람 리뷰어 코멘트 - 총 3개
| # | 파일 | 라인 | 내용 |
|---|------|------|------|
| 1 | Button.vue | 75 | :deep() 문법 사용 |
| 2 | Button.vue | 58 | === 변경 |
| 3 | api.ts | 107 | 스펙 변경 확인 |

어떤 항목을 수정할까요?

사용자: 1, 2번

AI: [1번 수정 + diff 제시]
컨펌해주세요.

사용자: 확인

AI: [2번 수정 + diff 제시]
컨펌해주세요.

사용자: 확인

AI: 모든 수정 완료. 커밋할까요?
```
